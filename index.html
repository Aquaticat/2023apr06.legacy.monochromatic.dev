Monochromatic - frontend designs

Introduction

Monochromatic aims to define some sensible sets of styling for text-heavy websites.
For example, a personal blog made of blog posts or a manual organized in chapters.

What is inside this repo?

There are several top-level directories in this repo.

  • idea
  • design
  • code

Top-level Directory idea

Documentation of design decisions.

A typical document would communicate:

  1. Which purposes the UI element should fulfill, and which purposes the UI element should fail to fulfill.
  2. How the UI element should be presented, and how the UI element should not be presented.
  3. The user interactions the UI element should be able to react to, and the user interactions the UI interface should not be able to react to.

Top-level Directory design

Implementation of design decisions in design software.

Top-level Directory code

Implementation of the result of design software.

Inside of those top-level directories, some subdirectories will be created for variants of "sensible set of styling"

  • Afzelia
  • Buforrestia
  • Calochortus
  • Draba

Subdirectory Afzelia

In this variant of "sensible set of styling", only one color except black and white is allowed.
Specifically hex #ff5e5e.

Choosing hex #ff5e5e is because it's the closest color to pure red that contrast against black enough to meet WCAG AAA guidelines.
This makes it possible to create a reasonable dark color scheme.

Subdirectory Buforrestia

hex #ff0000, pure red's transparency levels 12.5%, 25%, 37.5%, 50%, 62.5%, 75%, 87.5% are allowed to be used in this variant.

What this project is not.

The code written in this project is mainly used for demonstration.

I1 do not seek to write high-quality code. Therefore, lots of coding standards are not followed and lots of common linters are not applied. Expect lots of duplicated code and poor optimization.

However, due to the advance in modern devices' computing capacity, most of these code snippets should work well enough, if not faster, than using a heavyweight frontend framework. Therefore, I believe it's still important and meaningful to publish my implementations.
